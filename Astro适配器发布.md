我们隆重推出 EdgeOne Pages 的 Astro 全栈部署功能！现在，您可以在我们的平台上直接构建和部署完整的 Astro 应用程序，支持服务器端渲染、静态站点生成和混合模式。无论是内容驱动网站、博客还是复杂的 Web 应用，Pages 都能高效处理，助您轻松实现全栈开发。

## 新的全栈功能

在本次更新中，我们推出了两个核心功能：

### 1. Astro 全栈支持

EdgeOne Pages 现在提供全面的 Astro 支持，能够处理多种渲染模式，满足不同场景的需求。

- **渲染模式**：支持 SSR（服务器端渲染）、SSG（静态站点生成）和 Hybrid（混合模式），您可以在同一项目中针对不同页面采用不同的渲染策略，兼顾性能与灵活性。
- **基于文件的路由**：通过文件和文件夹结构自动生成路由，使页面管理变得简单直观。
- **内容集合**：支持 Markdown 和 MDX 内容管理，通过类型安全的内容集合 API 轻松管理内容。
- **多框架支持**：无缝集成 React、Vue、Svelte、Preact 等主流前端框架，在同一个项目中自由选择最适合的组件库。
- **API 路由**：内置 API 功能，可直接在 Astro 项目中处理 Web 请求，无需单独部署后端服务。

### 2. 一键部署与智能构建

Pages 为 Astro 项目提供零配置的一键部署体验，自动处理构建优化、依赖分析和资源分发，让您专注于代码开发。

- **零配置部署**：只需连接 Git , Pages 自动识别 Astro 项目，无需手动配置构建命令和输出目录，开箱即用。
- **智能构建优化**：构建时自动分析项目依赖，只打包运行时必需的文件，减少部署体积，提升启动速度。
- **自动资源分发**：静态资源自动通过 EdgeOne 全球 CDN 加速分发，动态页面在边缘节点就近渲染，实现最佳性能。
- **构建日志与监控**：在控制台实时查看构建日志和部署状态，快速定位问题，支持构建历史回滚。
- **环境变量管理**：支持在控制台配置环境变量，区分开发、测试、生产环境，无需修改代码即可切换配置。

## Pages 全栈开发优势

Pages 在多个方面简化了 Astro 全栈开发：

- **代码统一管理**：前端、API 路由和无服务器函数代码集中在同一仓库进行版本控制，通过代码提交自动触发构建部署，实现一体化开发流程。
- **无服务器设计**：您的应用程序按需运行并自动扩展。无需担心流量激增、服务器维护和管理。
- **性能优化**：内置智能依赖分析和代码优化，自动删除不必要的文件（如 source maps、TypeScript 声明文件、测试文件等），确保部署包精简高效。
- **框架兼容**：与 Astro 生态系统无缝协作，支持 React、Vue、Svelte 等主流框架。您可以在不重写代码的情况下迁移现有应用程序。
- **类型安全**：完整的 TypeScript 支持，提供类型安全的内容集合和 API 路由。
